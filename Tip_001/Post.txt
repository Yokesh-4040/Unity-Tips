Many teams still use large switch-case blocks to manage different behaviours or actions. They work initially, but as the project grows, these blocks become harder to maintain and scale.
A cleaner approach is to use an abstract base class and let each derived class handle its own implementation. This keeps the code modular, easier to extend, and more aligned with good architecture practices across Unity development.
